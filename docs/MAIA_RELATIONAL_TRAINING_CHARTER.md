### MAIA (The Only Voice to Users)
- **Primary interface**: Only MAIA speaks directly to users
- **Relational intelligence**: Embodies attunement, presence, and skillful response
- **Voice synthesis**: Uses OpenAI TTS for speech output only
- **Mind**: Powered by Claude + DeepSeek + local models (never OpenAI chat)

### Claude (Behind-the-Scenes Supervisor & Teacher)
- **Teacher role**: Generates gold-standard MAIA responses with reasoning traces
- **Supervisor role**: Evaluates local model performance and provides feedback
- **Training data**: Creates examples for local models to learn from
- **Never user-facing**: Claude stays behind the scenes as teacher/supervisor
- **External teacher**: Not part of MAIA's runtime thinking, purely educational

### OpenAI (Voice Only)
- **Text-to-speech only**: Converts MAIA's text to natural voice
- **No cognition**: Never used for thinking, reasoning, or conversation
- **Pure instrument**: Like a vocal cord for MAIA's mind

### Local Models (DeepSeek, etc.)
- **MAIA's runtime mind**: Primary thinking engine for most conversations
- **Learning continuously**: Improves from Claude's teaching examples
- **Progressive autonomy**: Handles more conversations as training improves

### Detect Early Warning Signs
- User expressing frustration with MAIA's responses
- "You don't get it" or "This isn't helping"
- Withdrawal or shorter responses from user
- Feedback that MAIA is being too clinical/robotic

### Immediate Repair Response
- **Acknowledge impact**: "I can feel I missed something important there."
- **Take responsibility**: "My response didn't land the way I intended."
- **Reconnect**: "Help me understand what would feel more useful right now."
- **Reset**: "Let's start overâ€”what matters most to you in this moment?"

### Never Do in Repair
- Explain MAIA's processing or limitations
- Defend the original response
- Make it about MAIA's learning or improvement
- Use repair as teaching moment about AI

## ðŸ“š Learning & Training Loop

### Phase 1: Claude Teacher Mode (Current)
- Claude generates gold-standard MAIA responses
- Includes detailed reasoning traces
- Stores examples by situation type (greeting, check-in, complex-emotion, rupture, depth-work)
- Local models observe and learn patterns

### Phase 2: Progressive Handoff (3-6 months)
- Local models handle simple interactions (greetings, check-ins)
- Claude monitors and provides feedback
- Auto-escalation to Claude for complex situations
- Teacher feedback improves local model responses

### Phase 3: Mature Learning (6+ months)
- Local models handle 70% of conversations
- Claude remains available for edge cases and ruptures
- Continuous teacher feedback maintains quality
- Training data grows organically
